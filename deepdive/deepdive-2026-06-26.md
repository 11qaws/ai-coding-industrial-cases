## Edge AI 기반 실시간 설비 이상탐지 (GNN 기반 경량 추론)
- **카테고리:** 생산기술
- **사례 수:** 2

### 개요
AIBIZ DutchBoy는 AMD EPYC 9355/9554 CPU 기반 Edge 서버를 Etch 장비 내부에 설치, 100ms 간격 300개 센서 데이터를 수집하여 GNN(Graph Neural Network) 기반 시계열 이상탐지 모델(<10만 파라미터)로 Arcing 현상을 사전 감지. McKinsey 사례에서는 RF 전력 0.3% 드리프트와 온도 0.5°C 변동을 400단계 전 감지. GPU 없이 CPU 전용 추론으로 Fab 내 발열/냉각 문제 해결. Docker 컨테이너 기반 아키텍처(20장비:1서버).

### 생산 기술 적용
- **공정:** Etch
  적용: Arcing 현상 실시간 모니터링 및 사전 감지 (DutchBoy, 300개 센서, 100ms 수집, 5분 추론 주기)
  효과: 수율 3~5% 향상, 연간 $100M 절감 (Fab당)

- **공정:** 확산
  적용: RF 전력/온도 드리프트 400단계 전 선제 감지 및 챔버 매칭 조기 경보
  효과: 공정 안정성 30% 이상 향상, 리드타임 단축

- **공정:** CMP
  적용: 플레턴 온도/압력 센서 시계열 기반 연마 종말점 이상 예측
  효과: 설비 가용성 10~20% 증가 (Deloitte 예지정비 효과)

### 운영 기술 적용
- **영역:** FDC
  적용: GNN 패턴 기반 다변량 이상 진단으로 단변량 SPC 한계 대체, False Positive 50% 이상 저감
  프로세스 변경: 기존 SPC 단변량 임계치 기반 알람 → GNN 기반 다변량 시계열 패턴 인식 + FDC 알람 연동

- **영역:** R2R
  적용: 장비 드리프트 사전 감지하여 R2R 보정 계수 Feed-forward 자동 보정
  프로세스 변경: R2R 피드백 주기에 이상탐지 Feed-forward 추가 → 보정 속도/정확도 향상

- **영역:** 스케줄링
  적용: 이상징후 장비 사전 식별로 정비 스케줄 최적화
  프로세스 변경: 정기 PM → 상태기반(CBM) 전환, 비가동시간 20~50% 단축

### 도입 패턴
- 장비 내부 Edge 서버 탑재 온프레미스 구조 (DutchBoy, 20장비:1서버, Docker)
- CPU-only 추론으로 GPU 발열/전력 부담 없이 기존 Fab 인프라에 통합
- 단일 장비 PoC → 라인 전체 확장 → 타 공정 전이 단계적 도입

### 성공 요인
- 10만 파라미터 미만 경량 모델로 Edge 하드웨어 실시간 추론 가능
- Docker 컨테이너 배포로 다양한 장비 플랫폼에 이식성 확보
- CPU-only 운영으로 NVIDIA GPU 의존도 및 라이선스 문제 회피
- 기존 5% 샘플링 검사 대신 전수 예측 모니터링으로 검사 비용 혁신적 절감

### 도전 과제
- 장비 기종별 상이한 센서 구성에 따른 모델 재학습/파인튜닝 필요
- 100ms 수집 데이터 저장/처리 인프라 구축 비용 (300채널 × 장비수 × 24/7)
- GNN 모델 해석 가능성(Explainability) 부족으로 엔지니어 신뢰 확보 과제
- AMD EPYC CPU 도입 시 기업 표준 하드웨어 정책과 충돌 가능성

### 주요 지표
- 수율(Yield): 3~5% 향상 (DutchBoy 사례)
- 연간 비용 절감: $100M/Fab (DutchBoy 사례)
- 설비 가용성(OEE): 10~20% 증가 (Deloitte 예지정비 효과)
- 검사 전환: 5% 샘플링 → 전수 예측 모니터링
- 추론 주기: 5분 (100ms 수집 데이터)

### 내부 배포 검토
- **내부망 가능성:** 높음 — Edge 서버가 장비 내부에 위치하여 데이터 외부 반출 불필요, CPU-only 추론으로 폐쇄망 내 자체 구축 가능, Docker 컨테이너 기반 내부 레지스트리 배포 가능
- **데이터 요구사항:** 장비 센서 데이터 (RF 전력, 온도, 압력, 가스 유량 등, 장비당 300+ 채널), 이상/정상 레이블 데이터 (최소 6개월), 장비 이력 및 PM 데이터
- **통합 복잡도:** medium
- **예상 기간:** 12개월

### 제안
- Etch 장비 Arcing 감지 PoC로 선정 (DutchBoy 검증 완료, 수율 효과 입증)
- AMD EPYC 기반 Edge 서버 도입 시 표준 하드웨어 예외 프로세스 선행 수립
- 10만 파라미터 미만 경량 모델 설계 원칙 수립으로 추론 효율성 보장
- FDC 시스템과 GNN 추론 결과 연동 API 표준화 추진

## ViT/딥러닝 기반 자동 결함 분류 (ADC)
- **카테고리:** 생산기술
- **사례 수:** 3

### 개요
IBM Albany 300mm Fab에서 DinoV2(ViT, Vision Transformer) 기반 self-supervised transfer learning을 SEM ADC에 적용, 11개 결함유형 7,400+장 이미지 분류에서 클래스당 15장 미만 소량 데이터로 90% 이상 분류 정확도 달성. Applied Materials 2세대 CFE e-beam 결함검사에 AI 알고리즘 통합으로 수동 리뷰 작업량 50% 감소 및 False Positive 저감. TSMC는 Deep Learning 기반 불량검출을 7nm/5nm Fab에 적용, 불량률 40% 감소 및 수율 20% 향상.

### 생산 기술 적용
- **공정:** Photo
  적용: EUV 패턴 결함 SEM 이미지 자동 분류 (DinoV2, 11개 유형, 90%+ 정확도)
  효과: 수동 리뷰 50% 감소, 분류 시간 90%+ 단축 (8시간→30분)

- **공정:** Etch
  적용: Plasma Etching 프로파일 결함(notching, bowing) 자동 식별 및 심각도 분류
  효과: False Positive 50% 저감, 엔지니어 핵심 이슈 집중

- **공정:** CMP
  적용: 스크래치/오염 입자 결함 분류 및 발생 원인 패턴 분석 연계
  효과: 불량률 40% 감소 (TSMC 7nm/5nm 사례)

- **공정:** 박막
  적용: 박막 결함(pin-hole, delamination) SEM 이미지 기반 자동 분류
  효과: 수율 20% 향상 (TSMC 사례)

### 운영 기술 적용
- **영역:** ADC
  적용: Rule-based ADC → ViT Self-supervised 학습 기반 분류로 전환, 신규 결함 유형 자동 적응
  프로세스 변경: Rule 엔지니어링 → Feature embedding 유사도 분류 + Semi-supervised learning, 재학습 주기 80% 단축

- **영역:** 레시피
  적용: 결함 분류 결과와 공정 레시피 파라미터 간 상관분석으로 결함 유발 조건 자동 식별
  프로세스 변경: ADC 결과를 Recipe Management System 피드백 → 최적 레시피 도출 Loop 구축

- **영역:** FDC
  적용: ADC 이상 결함 증가 알람과 FDC 센서 이상 패턴 교차 분석으로 근본 원인 추적
  프로세스 변경: ADC + FDC 이중 검증 체계로 False Positive 저감 및 신뢰도 향상

### 도입 패턴
- In-house Platform-agnostic ADC Tool 구축 (DinoV2 방식, 장비/공정 독립적)으로 벤더 종속성 제거
- 소량 데이터(클래스당 15장 미만) 학습으로 초기 라벨링 비용 최소화 → 점진적 데이터 증강
- 단일 결함 유형 PoC → 전체 결함 유형 확대 → 타 공정/장비군 전이

### 성공 요인
- Self-supervised 사전학습(ViT) + 전이학습으로 소량 데이터로도 높은 일반화 성능
- Platform-agnostic 설계로 장비 교체/추가 시 재개발 최소화
- Class 증분 학습(Incremental Learning) 지원으로 신규 결함 유형 지속 추가 가능

### 도전 과제
- DinoV2/ViT는 GPU 추론 필요, 기존 CPU-only 검사 장비에 GPU 탑재 또는 중앙 GPU 서버 구축 필요
- 세분화된 결함 유형 정의 및 레이블링 표준화 필요 (현행 수동 분류와 정합성)
- Self-supervised 모델 Black-box 특성으로 설비/공정 엔지니어 분류 결과 신뢰도 확보 과제
- SEM 이미지 해상도/포맷 차이로 Domain Adaptation 추가 개발 필요

### 주요 지표
- 분류 정확도: 90%+ (클래스당 15장 미만 학습, DinoV2 사례)
- 수동 리뷰 감소율: 50% (Applied Materials 사례)
- 불량률 감소: 40% (TSMC 사례)
- 수율 향상: 20% (TSMC 사례)
- 분류 시간 단축: 8시간 → 30분 (수동 대비 90%+ 단축)

### 내부 배포 검토
- **내부망 가능성:** 중간 — Self-supervised 학습(ViT)으로 외부 데이터 의존도 낮으나 사전학습 모델(DinoV2) 내부망 반입 필요, GPU 서버 내부 구축 가능, NVIDIA GPU 수급 및 CUDA 환경 구성 필요
- **데이터 요구사항:** SEM/광학 검사 이미지 (7,400장+), 결함 유형 레이블 (11개 유형, 클래스당 15장 이상), 검사 장비 메타데이터 (해상도, 배율, 검사 조건)
- **통합 복잡도:** medium
- **예상 기간:** 14개월

### 제안
- DinoV2 기반 SEM ADC PoC를 Photo 공정(EUV 결함)으로 시작, IBM Albany 사례 벤치마킹
- GPU 서버 공용 Pool 구축으로 검사 장비별 GPU 탑재 부담 완화 (추론 API 중앙 집중화)
- 결함 유형 표준 분류 체계(Taxonomy) 선행 수립, 현행 엔지니어 분류 프로세스와 정합성 확보

## 가상계측 (Virtual Metrology, ML 기반 공정 결과 예측)
- **카테고리:** 생산기술
- **사례 수:** 2

### 개요
McKinsey 분석에 따르면 ML 기반 가상계측(Virtual Metrology)으로 물리적 검사 빈도를 50~70% 감소, 사이클타임 10~15% 개선 가능. TSMC는 Deep Learning 기반 예측 모델을 7nm/5nm Fab 불량검출에 적용하여 수율 20% 향상. 구체적 알고리즘: LSTM/CNN 기반 시계열 예측, XGBoost/LightGBM 회귀 모델. 가상계측은 실측(CD-SEM, Ellipsometer)을 대체하여 생산성 및 비용 측면 이점 제공.

### 생산 기술 적용
- **공정:** CMP
  적용: 플레턴 온도/압력/시간/슬러리 유량 기반 Removal Rate 실시간 예측 (ML 회귀 모델)
  효과: 검사 빈도 50~70% 감소, 사이클타임 10~15% 개선

- **공정:** Etch
  적용: RF 전력/가스 유량/압력/온도 기반 Etch Rate 및 Profile 예측 (LSTM 시계열)
  효과: CD 측정 주기 연장, 장비 가동률 향상

- **공정:** 박막
  적용: 증착 시간/온도/전구체 유량 기반 박막 두께 및 균일도 예측
  효과: 측정 설비(Thickness Gauge) 부하 50% 감소

- **공정:** Photo
  적용: 노광량/초점/레티클 정렬 데이터 기반 CDU 예측
  효과: EUV CDU 모니터링 비용 절감, 수율 2~5% 추가 확보

### 운영 기술 적용
- **영역:** R2R
  적용: 가상계측 결과 R2R 피드백 즉시 반영, 피드백 지연시간 90% 단축
  프로세스 변경: 실측 기반 피드백(1~3시간 지연) → VM 예측 Feed-forward(실시간), 보정 주기 60분→1분 수준

- **영역:** FDC
  적용: VM 예측값과 실측값 간 잔차(Residual) 모니터링을 FDC 알람 보조 지표로 활용
  프로세스 변경: FDC 단변량 SPC + VM 잔차 다변량 모니터링 이중 검증 체계

- **영역:** 스케줄링
  적용: VM 결과로 검증 대체 가능 Lot 우선 스케줄링 (측정 장비 병목 완화)
  프로세스 변경: Lot별 VM 신뢰도 기반 측정면제 판정 로직 도입 → 측정 Queue 50%+ 감소

### 도입 패턴
- 특정 공정/레시피 VM 모델 개발 → R2R 연동 검증 → 타 공정/장비 확장
- 기존 측정 데이터 기반 지도학습 → 생산 적용 중 추가 데이터로 모델 점진적 고도화
- VM Confidence Score 기반 Lot Release 결정 (High Confidence → 측정면제, Low → 실측)

### 성공 요인
- 설비 센서 + 계측 데이터 정합성 확보가 가장 중요 (시점 정렬, Pre-processing)
- Confidence 출력 기반 신뢰도 의사결정 프레임워크로 초기 리스크 관리
- 기존 R2R/APC 시스템과 API 통합으로 검증 사이클 단축

### 도전 과제
- 공정 조건 변화(레시피 변경, 챔버 매칭, 장비 노화)에 따른 Concept Drift → 재학습 주기 관리
- 실측 없이 VM만 의존시 드리프트 누적 위험 (주기적 실측 검증 필수)
- 초기 학습 위한 충분한 정합 데이터 수집 및 전처리 파이프라인 구축 필요 (3~6개월)
- VM 예측값의 설비/공정 엔지니어 수용도 및 의사결정 권한 위임 이슈

### 주요 지표
- 검사 빈도 감소: 50~70% (McKinsey 사례)
- 사이클타임 개선: 10~15% (McKinsey 사례)
- 수율 향상: 2~5% (추가 확보)
- R2R 피드백 지연시간: 1~3시간 → 실시간 (VM 연동 시)
- 측정 장비 부하 감소: 50%+

### 내부 배포 검토
- **내부망 가능성:** 높음 — 모든 데이터 Fab 내부 생성/소비, ML 학습/추론 전 과정 내부망 가능, 오픈소스 ML 라이브러리(scikit-learn, XGBoost, PyTorch) 내부 환경 구성
- **데이터 요구사항:** 설비 센서 데이터 (RF 전력, 온도, 압력, 가스 유량, 1sec~1min Log), 계측 데이터 (CD-SEM, Ellipsometer, Thickness), 공정 레시피, R2R 보정 이력
- **통합 복잡도:** high
- **예상 기간:** 18개월

## GPU 가속 Computational Lithography (cuLitho 기반 OPC/HMI 가속)
- **카테고리:** 생산기술
- **사례 수:** 3

### 개요
NVIDIA cuLitho를 TSMC 마스크 설계 Computational Lithography에 적용하여 CPU 대비 Cost Effectiveness/Cycle Time 20~50% 개선. cuLitho는 GPU 가속 라이브러리로 OPC/ILT/마스크 디자인 최적화를 NVIDIA H200 GPU에서 병렬 처리. ### 생산 기술 적용
- **공정:** Photo

- **공정:** Photo
  적용: EUV Lithography 광학 수차 실시간 보정 AI 모델 (sub-3nm GAA 패턴)
  효과: CDU 10%+ 개선, 수율 2~5% 향상

- **공정:** Photo
  적용: cuEST 기반 전자구조 시뮬레이션으로 차세대 PR 소재 설계 가속

## ML 기반 멀티변수 APC/R2R (Advanced Process Control)
- **카테고리:** 운영기술
- **사례 수:** 2

### 개요
McKinsey 분석에 따르면 ML 기반 멀티변수 최적화 APC로 수율 2~5% 추가 향상 및 공정 Cpk 개선 가능. NVIDIA cuML GPU 가속 ML 라이브러리를 공정제어에 적용, 수백개 공정단계/수십만개 제조파라미터 분석으로 공정변동성 유의미한 감소. 기존 단변량 SPC/EWMA R2R 한계 극복, 파라미터 간 교호작용 고려한 다변량 최적 제어가 핵심. LSTM/Transformer 시계열 예측을 Feed-forward 제어에 결합하여 보정 정확도 향상.

### 생산 기술 적용
- **공정:** CMP
  적용: 패드 수명/슬러리 농도/연마 시간 다변량 최적화 R2R (ML 기반 Removal Rate 예측 + Feed-forward)
  효과: 수율 2~5% 향상, Cpk 20% 개선

- **공정:** Etch
  적용: RF 전력/가스비/압력/온도 다변량 최적 제어로 Etch Rate/CD 균일도 동시 보정
  효과: CD Uniformity 15% 개선, 불량률 30% 감소

- **공정:** 박막
  적용: 증착 온도 Profile/전구체 유량/시간 다변량 최적화로 두께/균일도/응력 동시 제어
  효과: 박막 균일도 10% 개선, Cpk 25% 향상

- **공정:** Photo
  적용: 노광량/초점 Offset 다변량 R2R + EUV 광원 세기 드리프트 Feed-forward 보상
  효과: CDU 15% 개선, 수율 3% 향상

### 운영 기술 적용
- **영역:** R2R
  적용: EWMA 단변량 R2R → ML 다변량 R2R 전환 (LSTM/Transformer 시계열 예측 + 다변량 최적화)
  프로세스 변경: EWMA 단변량 보정 → ML 다변량 Feed-forward+Feedback 동시 보정, 정확도 30%+ 향상

- **영역:** FDC
  적용: FDC 이상패턴 감지 시 R2R 제어 파라미터 자동 조정 (이상유형별 최적 제어 전략 매핑)
  프로세스 변경: 이상 감지 → 수동 중단/조정 → 재개 → ML 기반 이상유형별 R2R 파라미터 자동 전환

- **영역:** 레시피
  적용: cuML 분석 기반 레시피 파라미터 공정변동 영향도 평가 → 최적 레시피 도출
  프로세스 변경: 엔지니어 경험 기반 레시피 튜닝 → ML 분석 기반 레시피 최적화 권장 시스템

### 도입 패턴
- 단일 공정/장비 EWMA → ML 다변량 R2R 시범 적용 → 전 공정 확장
- 기존 APC 프레임워크(Space, Galaxy 등) ML 모듈 추가 통합
- 오프라인 ML 학습/시뮬레이션 → 온라인 실시간 제어 전환 (검증 기간 필요)

### 성공 요인
- 기존 R2R 인프라 활용 점진적 업그레이드 가능 (GPU 서버 추가만으로 PoC)
- 수율 2~5% 추가 향상 (McKinsey)으로 ROI 명확
- cuML GPU 가속으로 수십만개 파라미터 실시간 분석
- LSTM/Transformer 시계열 예측이 공정 드리프트 대응에 효과적

### 도전 과제
- 실시간 제어 Loop에 ML 추론 통합 Latency 보장 (<1초)
- ML 모델 이상 입력 Robustness — 제어 특성상 Fail-Safe 필수
- 기존 APC/R2R 시스템(Space, Galaxy 등)과 통합 인터페이스 표준화 부재
- ML 과도한 보정(Overshoot) 위험 — 안전장치(Guard Rail) 설계 필수
- 다변량 모델 Black-box 특성으로 엔지니어 제어 신뢰도 이슈

### 주요 지표

- **통합 복잡도:** high
