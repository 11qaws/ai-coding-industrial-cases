## cuLitho GPU 가속 리소그래피 OPC
- **카테고리:** 생산기술
- **사례 수:** 2

### 개요
NVIDIA cuLitho는 GPU 가속 컴퓨팅(CUDA)을 Photo 공정 OPC(Optical Proximity Correction) 및 마스크 디자인에 적용하는 라이브러리로, CPU 기반 워크로드 대비 리소그래피 시뮬레이션 사이클타임을 20-50% 단축. TSMC는 cuLitho를 3nm/5nm 양산 Fab의 OPC 파이프라인에 통합하여 CPU 클러스터 대비 전력 효율 40% 이상 개선 및 마스크 제작 턴어라운드 타임 단축. NVIDIA H200 GPU 기반 cuLitho는 기존 CPU 전용 OPC 서버 팜을 GPU 클러스터로 대체하여 총소유비용(TCO) 30% 이상 절감 효과

### 생산 기술 적용
- **공정:** Photo
  적용: OPC(Optical Proximity Correction) 마스크 보정 연산을 GPU 병렬처리로 가속하여 CPU 클러스터 대비 20-50% 사이클타임 개선
  효과: Photo 공정 마스크 제작 리드타임 단축 및 해상도 한계 극복을 통한 수율 향상

- **공정:** Photo
  적용: 마스크 디자인 규칙 검증(MRC) 워크플로우에 GPU 가속 적용으로 기존 CPU 대비 동일 예산 내 더 복잡한 OPC 모델 연산 가능
  효과: 고NA EUV 및 멀티패터닝에서 OPC 정밀도 향상으로 극한 해상도 공정의 수율 안정화

- **공정:** Etch
  적용: etch bias 보정을 위한 역전파 OPC 모델에 cuLitho 연산 활용 — etch loading 효과를 OPC 단계에서 사전 보정
  효과: Etch 공정 변동성 감소로 CD 균일도 10-15% 개선

### 운영 기술 적용
- **영역:** 레시피
  적용: OPC 레시피 파라미터 최적화에 GPU 병렬탐색 적용 — 수천개 OPC 파라미터 조합 실시간 평가
  프로세스 변경: 기존 CPU 배치처리 방식에서 GPU 실시간 최적화로 전환; 레시피 생성/검증 워크플로우 재설계 필요

- **영역:** FDC
  적용: cuLitho OPC 출력과 FDC 센서 데이터 상관분석으로 Photo 공정 이상 조기 감지 모델 구축
  프로세스 변경: FDC 수집 항목에 OPC 시뮬레이션 메트릭(edge placement error, MEEF) 추가; GPU 서버와 FDC DB 간 데이터 파이프라인 구축

### 도입 패턴
- TSMC가 NVIDIA와 공동으로 cuLitho 라이브러리를 Photo 공정 라인에 직접 통합 — 자체 OPC 엔진 대신 GPU 가속 생태계로 전환
- Fab 내부에 NVIDIA H200 GPU 클러스터를 온프레미스로 구축하여 OPC 워크로드 전용 처리; 클라우드 없이 완전 내부망 운영
- 기존 CPU OPC 서버 팜을 단계적으로 GPU 클러스터로 대체; 초기에는 critical layer OPC만 적용 후 모든 레이어로 확장

### 성공 요인
- NVIDIA CUDA-X 생태계의 성숙도 — cuLitho 라이브러리가 OPC 업계 표준(Calibre, Sentaurus)과 호환되어 기존 워크플로우 대체 부담 최소화
- TSMC의 강력한 GPU 인프라 투자 의지 — OPC 전용 GPU 클러스터에 $500M+ 투자하여 대규모 병렬처리 인프라 확보
- OPC 사이클타임 단축이 Photo 공정 전체 수율 학습 사이클에 직접적 선순환 효과: 빠른 OPC → 빠른 마스크 검증 → 빠른 수율 램프
- NVIDIA와의 공동 엔지니어링 계약으로 cuLitho 커스터마이징 및 TSMC 공정 특화 튜닝 가능

### 도전 과제
- CUDA 및 NVIDIA GPU에 대한 vendor lock-in 우려 — AMD ROCm 또는 Intel oneAPI 기반 대안 부재
- OPC 결과의 물리적 정확성 검증 필요: GPU 가속 근사해가 CPU full-physics 시뮬레이션 대비 0.1nm 이하 오차 허용범위 내인지 입증 필수
- 기존 OPC 엔지니어 인력의 GPU 프로그래밍(CUDA) 역량 부족 — 별도 교육/채용 필요
- Fab 내 GPU 클러스터 냉각 및 전력 인프라(Cooling Capacity, Power Distribution) 추가 소요

### 주요 지표
- OPC 사이클타임 단축률: 20-50%
- GPU 클러스터 TCO 대비 CPU TCO 절감율: 30% 이상
- edge placement error (EPE) 3σ 개선: 0.3-0.5nm
- Critical Layer Mask 제작 Lead Time 단축: 2-3일 → 1일 이내
- H200 GPU 1대당 CPU 서버 50-100대 대체 효과

### 내부 배포 검토
- **내부망 가능성:** 완전 내부망(on-premise) 배포 가능. NVIDIA GPU + CUDA cuLitho 라이선스를 볼륨 계약으로 온프레미스 설치. 네이티브 바이너리 배포로 외부 API 호출 불필요. 유일 리스크는 NVIDIA 드라이버 및 cuLitho 업데이트 시 외부 저장소 접근 필요 — Fab 전용 내부 패키지 미러 서버로 해결 가능
- **데이터 요구사항:** OPC 설계 데이터(GDSII/OASIS), 마스크 제작 레시피, Photo 공정 파라미터(NA, sigma, illumination 조건). 모든 데이터는 Fab 내부 DB 및 설계 툴 체인에서 획득 가능
- **통합 복잡도:** medium
- **예상 기간:** 18개월
